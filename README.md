# UnscrambleGame

Игра - угадайка, где вам нужно будет угадать все 20 слов и ввести их в строку чтобы получить баллы.
Но не так все просто - в каждом слове размешаны все буквы и вам дается всего одна попытка чтобы отгадать слово иначе вы теряете это слово навсегда

## Технологии

- Kotlin
- Jetpack Compose
- ViewModel
- StateFlow
- unit-test

## Функционал

- Вывод 20 различных перемешанных слов
- ввод слова в строку
- получение баллов
- возможность сыграть еще раз в конце


## Контрольные вопросы

1) Это класс который хранит и управляет данными UI, даже когда было пересоздание
   1) Сохраняет данные при повороте экрана
   2) Чистый код + тесты
   3) При повороте без него Activity пересоздается
2) MutableStateFlow - можно изменять значения, а StateFlow только читать
   1) Для того чтобы обесопаситься от нежелательного вмешательства в состояние
   2) Что такое “backing property pattern”? - это приватное изменяемое поле + публичная читательная версия
3) Потому что ViewModel дольше существует чем Activity
   1) Будет плохая ссылка с недейстивительным активити
4) remember - пропадает при повороте экрана, не переживает смерть процесса, rememberSaveable - ограничен  простыми типами, StateFlow - живет дольше, сохраняется при повороте  
   1) remember для временного состояния, rememberSaveable для поворота, StateFlow для сложных данных
5) Это класс для хранения данных
   1) toString, equals,hashCode,copy
   2) Помогает изменять состояние
6) Compose подписывается на состояние StateFlow
   1) потому что Compose подписа на изменения состояния
7) Для чистоты архитектуры, вохможности расширения и тестируемости кода
   1) Чистота архитектуры, Разделение логиеских компонентов
8) Это смерть процесса
   1) Данные уничтожаются
   2) Бд, возможно кеш
